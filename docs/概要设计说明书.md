# 票据审核系统 — 概要设计说明书

## 1. 引言

### 1.1 编写目的

依据需求规格说明书，从系统整体架构、模块划分、数据组织、接口规范等层面进行概要设计，为详细设计提供框架指导。

## 2. 系统架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    前端 (Vue 3 SPA)                   │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ ┌─────────┐ │
│  │  登录/   │ │ 我的票据 │ │ 票据审核  │ │ 系统管理 │ │
│  │  注册    │ │ (用户端)  │ │ (管理端)  │ │ (复用)   │ │
│  └────┬────┘ └────┬─────┘ └────┬─────┘ └────┬────┘ │
│       └───────────┴────────────┴─────────────┘       │
│                         │ Axios + JWT Token           │
└─────────────────────────┼─────────────────────────────┘
                          │ /dev-api (Vite Proxy)
                          ▼
┌─────────────────────────────────────────────────────┐
│                   后端 (Spring Boot)                  │
│  ┌───────────┐                                       │
│  │  Filter   │  JWT 认证 → SecurityContext           │
│  ├───────────┤                                       │
│  │ Controller│  @PreAuthorize 权限校验                │
│  ├───────────┤                                       │
│  │  Service  │  业务逻辑 (状态流转 / 权限过滤)         │
│  ├───────────┤                                       │
│  │  Mapper   │  MyBatis-Plus (无 XML / Lambda 查询)   │
│  └───────────┘                                       │
└─────────────────────────┼─────────────────────────────┘
                          │
             ┌────────────┼────────────┐
             ▼            ▼            ▼
        ┌────────┐  ┌─────────┐  ┌─────────┐
        │ MySQL  │  │  Redis  │  │ 磁盘文件 │
        │ 业务数据│  │Token/缓存│  │ 附件存储 │
        └────────┘  └─────────┘  └─────────┘
```

### 2.2 分层说明

| 层 | 技术 | 职责 |
|----|------|------|
| 表示层 | Vue 3 + Element Plus | 页面渲染、用户交互、数据展示 |
| 通信层 | Axios | HTTP 请求封装、Token 注入、防重复提交 |
| 服务层 | Spring Boot MVC | REST API、参数校验、权限拦截 |
| 业务层 | Service + ServiceImpl | 业务规则、状态流转、事务管理 |
| 数据层 | MyBatis-Plus | 无 XML 的 ORM 映射、Lambda 链式查询 |
| 存储层 | MySQL + Redis + 磁盘 | 结构化数据、会话缓存、文件存储 |

## 4. 数据库设计

### 4.1 核心表

| 表 | 说明 |
|----|------|
| biz_bill | 票据主表 |
| biz_bill_file | 附件表（票据 1:N 附件） |
| biz_audit_log | 审核记录表（票据 1:N 审核，完整历史） |
| biz_category | 票据类别表（已有） |
| sys_user / sys_role / sys_menu | 系统表（复用 RuoYi 框架） |

### 4.2 biz_bill 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint PK | 主键 |
| bill_no | varchar(32) UNIQUE NOT NULL | 草稿: DRAFT- 临时编号; 提交: BILL-YYYYMMDD-NNNN |
| title | varchar(255) | 票据标题 |
| category_id | bigint | 类别ID，FK → biz_category |
| amount | decimal(10,2) | 金额 |
| status | char(1) | 0-草稿 1-已提交 2-已通过 3-已退回 |
| create_by | varchar(64) | 创建者用户名（RuoYi 约定） |
| audit_by | varchar(64) | 审核人用户名 |
| audit_time | datetime | 审核时间 |
| audit_comment | varchar(500) | 审核意见 |

### 4.3 ER 关系

```
sys_user ──1:N── biz_bill ──1:N── biz_bill_file
                         ──1:N── biz_audit_log
           biz_bill ──N:1── biz_category
```

## 6. 权限设计

### 6.1 权限字符串

```
biz:bill:list    ← 菜单权限，进入「我的票据」或「票据审核」页面
biz:bill:query   ← 按钮权限，查看票据详情
biz:bill:add     ← 按钮权限，新增/修改/提交票据
biz:bill:remove  ← 按钮权限，删除草稿票据
biz:bill:review  ← 按钮权限，审核通过/退回
```

### 6.2 角色-权限矩阵

| 权限 | admin | 普通用户 |
|------|:-----:|:--------:|
| biz:bill:list | 有 | 有 |
| biz:bill:query | 有 | 有 |
| biz:bill:add | 无 | 有 |
| biz:bill:remove | 无 | 有 |
| biz:bill:review | 有 | 无 |

### 6.3 三级权限控制

| 级别 | 机制 | 示例 |
|------|------|------|
| 接口级 | `@PreAuthorize("@ss.hasPermi('biz:bill:add')")` | 后端 Controller 方法 |
| 按钮级 | `v-hasPermi="['biz:bill:add']"` | 前端按钮显示/隐藏 |
| 数据级 | Service 层过滤 `createBy = 当前用户名` | 普通用户只能看自己的 |

## 7. 接口设计概要

| Method | Path | 权限 | 请求体 | 响应 |
|--------|------|------|--------|------|
| POST | `/api/bill/query` | bill:list | BizBillQueryDto | PageResult |
| GET | `/api/bill/{id}` | bill:query | — | BizBillDetailVo |
| POST | `/api/bill` | bill:add | BizBillDto | — |
| PUT | `/api/bill` | bill:add | BizBillDto | — |
| POST | `/api/bill/submit/{id}` | bill:add | — | — |
| DELETE | `/api/bill/{id}` | bill:remove | — | — |
| POST | `/api/bill/review` | bill:review | BizBillReviewDto | — |

统一响应格式（ResultVo）:

```json
{ "code": 200, "message": "success", "data": {} }
```

## 8. 关键技术决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| ORM 查询 | MyBatis-Plus Lambda 查询 | 无需写 XML，类型安全 |
| 分页方案 | 模块独立 BasePageReq + PageResult | 与 BizCategory 模板一致 |
| 响应格式 | ResultVo（非 AjaxResult） | 与 zsc-module 模板统一 |
| 数据级权限 | Service 层判断 `hasPermi("biz:bill:review")` | 管理员看全部，用户只看自己的 |
| 附件存储 | 本地磁盘 + 数据库记录路径 | 复用 CommonController.upload |
| 票据编号 | 草稿: DRAFT- 临时编号; 提交: BILL-yyyyMMdd-NNNN + UNIQUE | 满足 NOT NULL，提交时替换 |
| 用户标识 | create_by 存用户名（varchar） | 遵循 RuoYi 框架约定 |

## 9. 开发约束

- 新模块放在 `zsc-module` 下，不修改框架核心代码
- 后端遵循 BizCategory 模板的包结构和命名规范
- 前端遵循 BizCategory 的组件通信模式（props-down, events-up）
- 不引入新的第三方依赖，复用框架已有的技术栈
