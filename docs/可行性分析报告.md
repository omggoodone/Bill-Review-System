# 票据审核系统 — 可行性分析报告

## 1. 引言

### 1.1 编写目的

从技术、经济、操作、时间四个维度评估票据审核系统项目的可行性，为是否进入设计阶段提供决策依据。

### 1.2 项目概述

本项目在 RuoYi-Vue 快速开发框架基础上，进行票据审核业务模块的二次开发。目标为组织内部员工提供在线票据提交与审核功能。

---

## 2. 技术可行性

### 2.1 技术栈分析

| 层级 | 技术 | 成熟度 | 说明 |
|------|------|--------|------|
| 后端框架 | Spring Boot 3.5.8 | 业界标准，版本成熟 | 已有完整框架搭建完毕 |
| ORM | MyBatis-Plus 3.5.9 | 国内主流 | 已配置，BizCategory 模块已验证可用 |
| 数据库 | MySQL 8.0 | 老牌 RDBMS | 已建库并导入 31 张表 |
| 缓存 | Redis | 成熟可靠 | 框架已集成，用于 Token/Session 管理 |
| 认证 | JWT + Spring Security | 行业标准 | 已实现完整登录鉴权流程 |
| 前端框架 | Vue 3.5 + Vite 5 | 主流前端框架 | 已配置 Element Plus + Pinia |
| 文件存储 | 服务器本地磁盘 | 简单直接 | CommonController 已实现上传/下载 |

**结论：全部技术均有成熟生态，无技术风险。** 框架提供的代码生成器可自动生成 80% 的 CRUD 代码，大幅降低开发工作量。

### 2.2 关键技术验证

| 技术点 | 是否可行 | 验证方式 |
|--------|----------|----------|
| 多角色权限控制 | 可行 | 框架已有 @PreAuthorize + v-hasPermi 三级权限 |
| 文件上传 | 可行 | CommonController 已有 upload 接口 |
| 状态流转 | 可行 | BizCategory 模块已验证实体 CRUD 流程 |
| 票据编号生成 | 可行 | MyBatis-Plus lambdaQuery 查最大号 + UNIQUE 约束 |
| 动态菜单 | 可行 | 框架已有 getRouters 接口 + import.meta.glob |

### 2.3 开发环境

| 工具 | 用途 | 风险 |

| VS Code / WebStorm | 前端开发 IDE | 无 |

## 4. 操作可行性

### 4.1 用户分析

- **管理员**：使用系统管理后台（Web 界面），操作方式为标准表单、按钮、表格，无需培训
- **普通用户**：仅需填写表单和上传文件，界面简洁，与日常办公系统类似

### 4.2 运行环境

- **用户端**：浏览器即可，无需安装客户端软件
- **服务端**：JDK 17 + MySQL 8.0 + Redis，均为常见企业环境
- **部署方式**：单 jar 包运行，前后端分离部署

### 4.3 团队能力

- 组员具备 Java Web 开发基础知识
- 框架自带的 BizCategory 模块提供了完整开发模板
- 项目已有运行指南文档，环境搭建标准化

**结论：操作可行，学习曲线低。**
