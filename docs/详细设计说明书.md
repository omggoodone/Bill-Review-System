## 2. 数据库详细设计

### 2.1 biz_bill（票据表）DDL

```sql
DROP TABLE IF EXISTS `biz_bill`;
CREATE TABLE `biz_bill` (
    `bill_id`         bigint(20)    NOT NULL AUTO_INCREMENT COMMENT '票据ID',
    `bill_no`         varchar(30)   DEFAULT NULL COMMENT '票据编号（BILL-YYYYMMDD-NNNN）',
    `title`           varchar(200)  NOT NULL COMMENT '票据标题',
    `bill_type`       varchar(50)   DEFAULT NULL COMMENT '票据类型（expense=报销 travel=差旅 office=办公 other=其他）',
    `amount`          decimal(12,2) NOT NULL COMMENT '金额',
    `description`     text          DEFAULT NULL COMMENT '票据描述',
    `status`          char(1)       DEFAULT '0' COMMENT '状态（0=草稿 1=已提交 2=已通过 3=已退回）',
    `submit_user_id`  bigint(20)    NOT NULL COMMENT '提交人ID',
    `review_user_id`  bigint(20)    DEFAULT NULL COMMENT '审核人ID',
    `review_comment`  varchar(500)  DEFAULT NULL COMMENT '审核意见',
    `create_time`     datetime      DEFAULT NULL COMMENT '创建时间',
    `update_time`     datetime      DEFAULT NULL COMMENT '更新时间',
    `submit_time`     datetime      DEFAULT NULL COMMENT '提交时间',
    `review_time`     datetime      DEFAULT NULL COMMENT '审核时间',
    PRIMARY KEY (`bill_id`),
    UNIQUE KEY `uk_bill_no` (`bill_no`),
    KEY `idx_submit_user_id` (`submit_user_id`),
    KEY `idx_review_user_id` (`review_user_id`),
    KEY `idx_status` (`status`),
    KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='票据表';
```

### 4.4 修改票据

```
PUT /api/bill
```

校验逻辑：
```
1. 查现有票据 → 不存在则抛异常
2. IF 现有票据.status ≠ "0" → 抛异常 "只能编辑草稿状态的票据"
3. IF 现有票据.submitUserId ≠ 当前用户 → 抛异常 "只能编辑自己的票据"
4. 更新字段 + updateTime=now
5. 删除旧附件 + 保存新附件
```

### 4.5 提交草稿

```
POST /api/bill/submit/{billId}
```

校验逻辑：
```
1. 查现有票据 → 不存在则抛异常
2. IF status ≠ "0" → 抛异常 "只能提交草稿状态的票据"
3. IF submitUserId ≠ 当前用户 → 抛异常 "只能提交自己的票据"
4. 生成 billNo（如未生成）
5. status ← "1"
6. submitTime ← now
7. update 票据
```

### 4.6 删除票据

```
DELETE /api/bill/{billId}
```

校验逻辑：
```
1. 查现有票据 → 不存在则抛异常
2. IF status ≠ "0" → 抛异常 "只能删除草稿状态的票据"
3. IF submitUserId ≠ 当前用户 → 抛异常 "只能删除自己的票据"
4. 删除所有附件记录
5. 删除票据记录
```

### 4.7 审核票据

```
POST /api/bill/review
```

请求体：
```json
{
  "billId": 1,
  "status": "2",
  "reviewComment": "费用合理，同意报销"
}
```

校验逻辑：
```
1. 查现有票据 → 不存在则抛异常
2. IF status ≠ "1" → 抛异常 "只能审核已提交状态的票据"
3. IF reviewDto.status ≠ "2" AND ≠ "3" → 抛异常 "无效的审核状态"
4. status ← reviewDto.status
5. reviewUserId ← 当前用户ID
6. reviewComment ← reviewDto.reviewComment
7. reviewTime ← now
8. update 票据
```

## 6. 票据编号生成算法

```
generateBillNo():
    dateStr ← new SimpleDateFormat("yyyyMMdd").format(new Date())
    prefix ← "BILL-" + dateStr + "-"
    
    // 查询当日最大编号
    wrapper ← LambdaQueryWrapper<BizBill>()
    wrapper.likeRight(BizBill::getBillNo, prefix)
            .orderByDesc(BizBill::getBillNo)
            .last("LIMIT 1")
    latest ← this.getOne(wrapper, false)
    
    IF latest == null OR latest.billNo == null:
        seq ← 1
    ELSE:
        parts ← latest.billNo.split("-")
        seq ← Integer.parseInt(parts[2]) + 1
    
    RETURN String.format("BILL-%s-%04d", dateStr, seq)
```

并发安全：bill_no 字段有 UNIQUE 约束，万一两个用户同时提交导致编号冲突，MySQL 会抛 DuplicateKeyException，前端提示"提交失败，请重试"即可。

## 7. 错误码设计

| code | message | 触发场景 |
|------|---------|----------|
| 200 | success | 正常 |
| 500 | 服务器内部异常 | 未知错误 |
| 997 | 参数校验错误 | @Valid 校验失败 |
| 1999 | 数据重复 | 编号冲突等 |

业务异常通过 ServiceException 抛出，message 由业务层定义：
- "票据不存在！"
- "只能编辑草稿状态的票据！"
- "只能编辑自己的票据！"
- "只能提交草稿状态的票据！"
- "只能删除草稿状态的票据！"
- "只能审核已提交状态的票据！"
- "系统错误，票据保存失败！"
