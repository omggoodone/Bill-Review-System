# 需求规格说明

> 完整文档: [需求规格说明书.md](../../docs/需求规格说明书.md)

## 1. 项目背景

基于 RuoYi-Vue（Spring Boot + Vue 3）框架进行二次开发，实现一个支持多角色的在线票据提交与审核系统。目标用户为组织内部员工（提交票据）、票据审核员（审核票据）与系统管理员（管理用户、审核注册）。

## 2. 术语定义

| 术语 | 说明 |
|------|------|
| 票据 | 用户提交的报销/差旅/办公等凭证，包含文字信息和附件 |
| 草稿 | 票据的临时保存状态，可反复修改，可删除 |
| 提交 | 将草稿票据正式提交进入审核流程，提交后不可再修改 |
| 审核 | 审核员对已提交票据做出通过或退回的决定 |
| 注册审核 | 用户提交邮箱申请 → 系统管理员审批 → 生成初始账号密码 |
| 普通用户 | 只能查看和操作自己票据的用户，可提交票据等待审核 |
| 票据审核员 | 审核已提交票据（通过/退回），查看审核统计仪表盘 |
| 系统管理员 | 管理用户（启停/删除/重置密码）、管理票据类别、审批注册申请 |

## 3. 系统角色

| 角色 | role_key | 权限概述 | 获得方式 |
|------|----------|----------|----------|
| 普通用户 | `user` | 管理个人票据（增/删/改/查），查看个人仪表盘 | 注册申请 → 管理员审批 |
| 票据审核员 | `reviewer` | 审核票据（通过/退回），查看审核统计仪表盘 | 注册申请 → 管理员审批 |
| 系统管理员 | `admin_user` | 用户管理、类别管理、注册审批、系统概览 | 数据库预置/超级管理员分配 |

## 4. 功能需求

### 4.1 普通用户功能

#### FR-01 用户注册（审核制）

- 用户通过注册页面提交申请
- 输入: 邮箱、申请说明、选择申请角色（普通用户/审核员）
- 提交后进入待审核状态，管理员审批通过后生成初始账号密码
- 管理员在控制台查看生成的用户名（邮箱前缀）和随机密码，转交用户

#### FR-02 添加票据

- 用户填写票据信息并选择"保存草稿"或"直接提交"
- 必填: 票据标题、金额
- 选填: 票据类别、描述、附件（单文件）
- 保存草稿: status=0，billNo 为临时编号（DRAFT- + 时间戳），可后续编辑
- 直接提交: status=1，生成正式编号 BILL-YYYYMMDD-NNNN

#### FR-03 修改草稿票据

- 前置条件: 票据 status=0（草稿）或 status=3（已退回），且为本人票据
- 约束: 已提交/已通过的票据不可修改

#### FR-04 删除草稿票据

- 前置条件: 票据 status=0（草稿），且为本人票据
- 票据及关联附件一并删除
- 约束: 已提交的票据不可删除

#### FR-05 提交草稿票据

- 将草稿或已退回票据正式提交进入审核流程
- status: 0 或 3 → 1，生成正式票据编号
- 提交后不可撤回、不可修改

#### FR-06 查看已通过票据

- 用户查看自己已通过审核的票据列表及详情（status=2）

#### FR-07 查看未审核票据

- 用户查看自己已提交但尚未审核的票据列表及详情（status=1）

#### FR-08 查看个人所有票据

- 按状态分类展示: 全部 / 草稿 / 待审核 / 已通过 / 已退回
- 分页加载，默认每页 10 条

#### FR-09 票据查询

- 查询条件: 关键词（标题）、票据类别、日期范围
- 输出分页列表: 票据编号、标题、类别名称、金额、状态、创建时间
- 约束: 只能查看本人提交的票据

#### FR-10 查看票据详情

- 查看票据完整信息，含附件、审核历史

### 4.2 票据审核员功能

#### FR-11 审核票据

- 对已提交的票据做出审核决定
- 输入: 审核结果（通过/退回）、审核意见
- 通过: status → 2，记录 auditBy 和 auditTime
- 退回: status → 3，记录 auditBy 和 auditTime
- 前置条件: 票据 status=1（已提交）

#### FR-12 查看待审核票据列表

- 查看系统中所有 status=1 的票据
- 支持按提交人（模糊）、票据类别、金额范围筛选
- 积压超过 3 天的票据高亮标记

#### FR-13 审核统计仪表盘

- 待审核票据数量
- 今日审核通过/退回数量
- 审核通过率
- 积压预警（待审核超过 3 天）
- 最近审核记录

### 4.3 系统管理员功能

#### FR-14 管理概览

- 统计面板: 总用户数、普通用户数、审核员数、管理员数、总票据数、待审核票据数

#### FR-15 用户管理

- 查看所有用户（仅显示普通用户和审核员，过滤系统自带用户）
- 按用户名、角色、状态筛选
- 操作: 重置密码、启用/停用、删除用户
- 复用框架已有接口

#### FR-16 注册审批

- 查看待审核的注册申请列表
- 通过: 自动生成用户名（邮箱前缀）+ 随机 8 位密码，创建 SysUser 并分配角色，弹窗展示初始账号密码
- 拒绝: 填写审批意见，不创建用户

#### FR-17 票据类别管理

- 对 biz_category 表进行 CRUD
- 按类别名称、状态筛选

## 5. 票据状态与操作矩阵

| 状态 | 标识 | 可编辑 | 可删除 | 可审核 | 可查看 |
|------|------|:------:|:------:|:------:|:------:|
| 草稿 | 0 | 是 | 是 | 否 | 是 |
| 已提交 | 1 | 否 | 否 | 是 | 是 |
| 已通过 | 2 | 否 | 否 | 否 | 是 |
| 已退回 | 3 | 是 | 否 | 否 | 是 |

## 6. 非功能需求

### 安全性
- 所有接口需 JWT Token 认证，未登录不可访问
- 权限控制到按钮级: 普通用户不能看到管理员专属按钮
- 后端校验数据归属: 用户只能操作自己的票据
- 密码 BCrypt 加密存储

### 可用性
- 状态以不同颜色标签区分: 草稿(灰) / 已提交(蓝) / 已通过(绿) / 已退回(红)
- 操作结果有明确提示（成功/失败消息）

### 性能
- 票据列表分页加载，默认每页 10 条
- 附件上传单文件限制 10MB

### 可扩展性
- 票据类别通过 biz_category 表管理，可后续扩展
- 审核流程通过 biz_audit_log 表支持多级审核扩展

## 7. 数据需求

### 票据数据

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 票据编号 | 字符串 | 是 | 系统自动生成: BILL-YYYYMMDD-NNNN |
| 标题 | 字符串 | 是 | 最长 200 字符 |
| 票据类别 | 外键 | 否 | 关联 biz_category 表 |
| 金额 | 数字 | 是 | 精确到小数点后两位 |
| 描述 | 文本 | 否 | 长文本 |
| 状态 | 枚举 | 是 | 0-草稿 1-已提交 2-已通过 3-已退回 |
| 附件 | 文件列表 | 否 | 支持多个文件 |

### 附件数据

| 字段 | 类型 | 说明 |
|------|------|------|
| 文件名 | 字符串 | 原始文件名 |
| 文件路径 | 字符串 | 服务器存储路径 |
| 文件大小 | 数字 | 字节数 |
| 上传时间 | 日期时间 | 上传时刻 |

## 8. 约束与假设

1. 基于已有 RuoYi-Vue 框架，用户管理、登录认证、权限控制等复用框架
2. 服务器环境: JDK 17 + MySQL 8.0 + Redis
3. 浏览器环境: Chrome / Firefox / Edge 现代版本
4. 单次附件上传不超过 10MB
5. 票据审核为单级审核，不涉及多级审批流
6. 用户标识字段 `create_by` 使用 RuoYi 框架约定的用户名（varchar），不使用 userId

## 9. 验收标准

| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| AC-01 | 用户提交注册申请（邮箱+说明+选择角色） | 申请进入待审状态 |
| AC-02 | 管理员通过注册申请 | 生成账号密码弹窗，SysUser 创建 |
| AC-03 | 管理员拒绝注册申请 | 状态变更为已拒绝 |
| AC-04 | 用户新建票据，保存草稿 | status=0，可再次编辑 |
| AC-05 | 用户提交草稿票据 | status→1，生成编号，编辑按钮消失 |
| AC-06 | 用户删除草稿票据 | 票据及附件被删除 |
| AC-07 | 用户查看票据列表 | 仅显示本人的票据 |
| AC-08 | 审核员查看待审核票据 | 显示所有已提交的票据 |
| AC-09 | 审核员审核通过票据 | status→2 |
| AC-10 | 审核员审核退回票据 | status→3，含退回意见 |
| AC-11 | 退回后编辑并重新提交 | 可编辑，可重新提交 |
| AC-12 | 审核员仪表盘显示统计数据 | 待审数/今日审核/通过率/积压预警 |
| AC-13 | 普通用户访问审核页面 | 无权限，页面不显示 |
| AC-14 | 管理员按状态筛选用户 | 返回匹配状态的用户列表 |
| AC-15 | 管理员删除用户 | 软删除（del_flag=2） |
| AC-16 | 侧边栏 Logo 点击 | 按角色跳转对应概览页 |
| AC-17 | 未登录访问任意接口 | 返回 401 未授权 |
