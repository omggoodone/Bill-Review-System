# 批量提交票据编号重复冲突

## 问题描述

批量提交多个草稿票据时，后端报错：
```
Duplicate entry 'BILL-20260526-0008' for key 'biz_bill.uk_bill_no'
```

## 复现步骤

1. 新增多个草稿票据
2. 全选 → 批量提交
3. 后端报编号重复错误

## 原因分析

`generateBillNo()` 的旧实现：查询当日最大编号 → 序号 +1。批量提交时多个请求并发执行：

- 请求A：查询最大编号 → `BILL-...-0007` → 生成 `0008`
- 请求B：查询最大编号 → `BILL-...-0007`（相同）→ 生成 `0008`
- 两个请求同时 UPDATE → 唯一约束冲突

## 解决方案

改用票据主键 `id`（数据库自增，全局唯一）作为编号序号：

```java
// 旧：查最大编号 + 1
private String generateBillNo() {
    BizBill latest = this.getOne(...);
    int seq = Integer.parseInt(latest.getBillNo().split("-")[2]) + 1;
    return String.format("BILL-%s-%04d", dateStr, seq);
}

// 新：用主键ID
private String generateBillNo(Long billId) {
    return String.format("BILL-%s-%04d", dateStr, billId);
}
```

`addBill` 改为先保存再生成编号：
```java
bill.setBillNo("DRAFT-" + System.currentTimeMillis());
this.save(bill); // 先存，获取 id
if ("1".equals(dto.getStatus())) {
    bill.setBillNo(generateBillNo(bill.getId())); // 用 id 生成编号
    this.updateById(bill);
}
```

`submitBill` 改为传入 `bill.getId()`：
```java
bill.setBillNo(generateBillNo(bill.getId()));
```
